Self-defined: Web-based version control software

Group student projects, such as that in SWI, can suffer from problems of version control (Wilson, 1991)- different students making incompatible changes to files. The objective of this project would be to build a web-based tool to be used to overcome this. There are a number of existing tools and this project will build on the literature (e.g. CVS, SVN, SCCS: Rochkind, 1975).

There are different aspects and approaches to this kind of system. A simple approach is to assign roles to different team members and to assign different corresponding files to them. Thus, for instance, one member might be working on the front-end of a software system. They will have one or more files which constitute that module and they alone will be allowed write-access to those files. Such a system requires user authentication and management of the users' capabilities. An alternative approach is to keep track of the changes to files (deltas) so that any version can be recreated at any time. If different people are allowed to make simultaneous changes, though, they may be incompatible. In that case it may be necessary to have a check-in and -out mechanism to prevent parallel access.

One of the first objectives of this project would be to ascertain what kind of system would be appropriate for us in SWI.

The plan is specifically to build a web-based system.  The advantages of a web-based system are:

  • There is no need for software installation.
  • Users can access it from anywhere.
  • It will be easy to use.

Plan

  1. Literature review: finding out about existing systems and approaches.
  2. Requirements analysis - mainly with students.
  3. Initial design.
  4. Prototype implementation and testing.
  5. Re-design.
  6. High-fidelity prototype implementation.
  7. Evaluation
  8. Write-up

The high-fidelity prototype will be built using PHP and mySQL with JavaScript and possibly Ajax, as appropriate.
